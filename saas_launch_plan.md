# Vivagen AI: SaaS Launch Roadmap & Strategy

This document outlines the strategic plan to transition Vivagen AI from an MVP to a production-ready SaaS product.

## 1. Project Analysis (Current State)
Vivagen AI is a robust "AI-Powered Technical Interviewer" built with **Next.js**, **Supabase**, and **Google Gemini**.

### Core Flow:
1.  **Setup**: User enters candidate details (Role, Level, Stack).
2.  **Generation**: Gemini generates tailored questions (conceptual, scenario, practical).
3.  **Interview**: Simple UI for scoring (0-10) and note-taking.
4.  **Evaluation**: Gemini analyzes scores and notes to provide a hiring recommendation.

---

## 2. API & Accountability (Scaling & Limits)
You mentioned concerns about the API key and accountability. Here is how to handle growth:

### API Key Management
*   **Rate Limiting**: Currently, if many users hit the API simultaneously, you might face `429 Too Many Requests`. 
    *   *Solution*: Implement a "Queue" or "Retry with Exponential Backoff" in the backend. 
    *   *Upgrade*: Move from the "Free" Gemini tier to a "Pay-as-you-go" tier to increase RPM (Requests Per Minute) limits.
*   **Key Security**: Never expose the key on the frontend (already handled correctly via Next.js API routes).

### Accountability (Usage Tracking)
*   **Current State**: You have a `usage_tracking` table limiting users to 10 interviews/month.
*   **Recommended Update**: Switch to **"AI Credits"**.
    *   Instead of "Interviews per Month", give users credits (e.g., 100 Credits).
    *   Generating questions = 10 Credits.
    *   Generating a summary = 5 Credits.
    *   This allows more granular control and prepares for monetization.

---

## 3. Recommended SaaS Features
To make this "Launch Ready" and attractive to actual users (HR, Dev Leads):

### Level 1: "User Experience" (Quick Wins)
*   **[ ] Shareable Report Links**: Allow users to share a read-only link of the interview summary with their team.
*   **[ ] PDF Export**: One-click download of the candidate's performance report.
*   **[ ] Dashboard Analytics**: A "Manager View" to see all candidate scores in one place (visualized with charts).

### Level 2: "Advanced AI" (Value Add)
*   **[ ] Audio-to-Text**: Integrate speech-to-text so the interviewer doesn't have to type notes manually.
*   **[ ] Ideal Answer Reference**: Show the "Ideal Answer" to the interviewer for each question (generated by AI).
*   **[ ] Live Coding Integration**: A simple code editor where candidates can type, and AI evaluates the code.

### Level 3: "Business/Enterprise" (Revenue)
*   **[ ] Team Workspaces**: Organizations can invite multiple "Interviewer" members.
*   **[ ] Custom Company Knowledge**: Upload a company's internal tech stack or guidelines to train the AI for even better questions.
*   **[ ] White-labeling**: Recruitment agencies can put their own logo on the reports.

---

## 4. Monetization & Freemium Strategy
To maximize reach without losing money on API costs:

### The "Maximum Freemium" Model
*   **Free Tier (Cost-Zero approach)**:
    *   **5 Interviews total** (instead of 10/month) to entice commitment.
    *   Limited questions (max 5).
    *   No PDF exports.
    *   *Why*: Keeps your Gemini bill low while letting users "feel" the value.
*   **Pay-As-You-Go**:
    *   Let users buy "Credit Packs" (e.g., $10 for 50 Interviews).
    *   *Benefit*: No monthly commitment for users, easy entry point.
*   **Pro Subscription ($29/mo)**:
    *   Unlimited interviews.
    *   PDF Exports & Team sharing.
    *   Priority API access (faster generation).

## 5. Technical Next Steps (Refactoring)
1.  **Remove Hardcoded Admin**: Move `ADMIN_EMAIL` to an environment variable or a `user_roles` table in Supabase.
2.  **Stripe Integration**: Add a simple `/pricing` page and Stripe checkout to sell credits.

4.  **Error Handling**: Add better UI feedback if Gemini fails (e.g., "AI is currently busy, please try again in 30 seconds").
